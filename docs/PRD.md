# 產品需求文件 (PRD)

## 1. 專案概述

### 背景與動機
在日常生活中，許多人面臨收支管理不平衡的狀況，往往到了月底才發現超支。為了幫助個人更好地掌握資金流動情況、養成良好的理財習慣，開發一款簡單易用、功能明確的個人記帳簿系統是非常必要的。

### 目標用戶
需要理財、追蹤日常開銷的個人用戶。

### 核心價值主張
提供一個直覺的操作介面，讓用戶能快速記錄每一筆收入與支出，並透過即時的餘額統計與分類報表，輕鬆掌握自身財務狀況。

## 2. 功能需求

以下為系統的主要功能以及對應的使用者故事 (User Story)：

1. **新增收入**
   - 作為 **個人用戶**，我希望 **能夠輸入並儲存收入金額、來源及日期**，以便 **記錄我的實際進帳狀況**。
   
2. **新增支出**
   - 作為 **個人用戶**，我希望 **能夠輸入並儲存支出金額、分類與日期**，以便 **記錄我每一筆花費明細**。
   
3. **顯示餘額**
   - 作為 **個人用戶**，我希望 **首頁能直接顯示目前的總餘額（總收入 - 總支出）**，以便 **我隨時了解自己還有多少可用資金**。
   
4. **分類統計**
   - 作為 **個人用戶**，我希望 **能看到依據類別（如：食、衣、住、行）的支出統計結果**，以便 **分析我的消費習慣與了解大宗開銷項目**。
   
5. **歷史紀錄查詢**
   - 作為 **個人用戶**，我希望 **能夠瀏覽過去所有的收支明細列表**，以便 **核對或回顧過去的財務狀況**。

## 3. 非功能需求

### 技術限制
- 後端模組：Python Flask
- 模板引擎：Jinja2
- 資料庫系統：SQLite

### 效能與安全考量
- **效能**：系統頁面載入應迅速，收支紀錄能即時更新顯示餘額，提供無縫體驗。
- **安全確保**：所有的輸入欄位需進行基本驗證；查詢條件需避免 SQL Injection 風險（可考慮使用 SQLAlchemy 等 ORM）。

## 4. MVP 範圍

本專案採敏捷開發，第一版 (MVP - Minimum Viable Product) 範圍定義如下：

| 層級 | 功能描述 |
| --- | --- |
| **Must Have (必須具備)** | - 新增收入與支出介面<br>- 首頁顯示當前餘額<br>- 列出所有歷史紀錄 |
| **Should Have (應該具備)** | - 紀錄篩選與排序功能<br>- 基礎的支出圓餅圖分類統計 |
| **Nice to Have (有會更好)** | - 匯入與匯出 CSV 功能<br>- 各類別的預算設定及即將超支提醒 |
